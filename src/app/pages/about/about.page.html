About Tweetsicle

Technologies Used

  • SignalR
  • Angular4
  • RxJS
  • ngrx/store

Tweetsicle provides real a real time Twitter feed from Twitter's streaming API and serves this to an Angular client via SignalR. The user can provide a series of search terms, separated by spaces, and the API will return every tweet with hash tags that match the query.

For instance, try searching for "trump usa". You will now see a live stream of tweets that contain "#trump" OR "#usa", as well as a chart showing all OTHER hashtags used in those tweets.

Searching for a new term will clear the store and start streaming tweets that match the new query instead.

Pages

Feed

The Feed page displays each new tweet in real time. Clicking on a tweet will take you to the raw JSON data that was received from the API for that tweet.

Insights

The Insights page displays a chart of the top 10 hashtags in the matching tweets, filtering out the hashtags you searched for. It also displays the raw hashtag counts beneath the chart so you can verify the data.

Log

The log page displays the raw results that comes back from the API.

Known Issues

The API is fairly crude and untested. The TweetInvi library occasionally fails to generate the HTML for a tweet, and I am yet to find out why. When this happens I return a placeholder with the tweet text, which serves the same purpose but doesn't have links or images..

It can also get into a state where it returns a 503 response to the client. I usually fix this by restarting the app in Azure.

Although the app is designed to allow the user to change search terms, I have observed rare issues where the previous search term remains subscribed. Usually a page refresh will resolve this.

The client is not nearly as thoroughly tested as I would like, nor as thoroughly styled.
